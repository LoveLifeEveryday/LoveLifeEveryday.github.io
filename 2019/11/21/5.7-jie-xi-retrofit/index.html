5.7 解析Retrofit

什么是Retrofit？为什么要用Retrofit呢

1.什么是Retrofit？

Retrofit是基于OkHttp实现的，使用运行时注解的方式提供功能

2.基本用法

2.1 导入框架

implementation 'com.squareup.retrofit2:retrofit:2.0.2'
    implementation 'com.squareup.okhttp3:okhttp:3.4.1'  
 implementation 'com.squareup.retrofit2:converter-gson:2.0.2'  //返回值为gson数据
 implementation 'com.google.code.gson:gson:2.8.5'   //gson
implementation 'com.squareup.retrofit2:adapter-rxjava:2.0.2'  //rxjava

2.2 注解分类

2.2.1 HTTP请求方法注解

GET、POST、PUT、DELETE、 HEAD、PATCH、OPTIONS和HTTP（可以替换前面的）

2.2.2 标记类注解

  • FormUrlEncoded：标明这是一个表单请求，一般配合post请求使用
  • Multipart：允许多个@Part,一般是在文件上传的时候使用
  • Streaming：以流的形式返回，如果不使用它，则默认会把全部数据加载到内存，所以下载大文件时需要加上这个注解

2.2.3 参数类注解

有Header、Headers、Body、Path、Field、 FieldMap、Part、PartMap、Query和QueryMap等

3.GET请求

3.1 新建接口

//Call其实是返回的数据的集合
public interface UserMgrService {
    //get请求，参数用Query，@QueryMap是Query的集合
    @GET("login")   //192.168.106:8080/login?username=张三&pwd=123456,对应baseUrL后面的
    Call<UserInfoModel> login(@Query("username") String username,@Query("pwd") String pwd);

    //post请求的话，参数用Field
    @POST("login")
    @FormUrlEncoded   //URL编码
    Call<UserInfoModel> login(@Field("username") String username, @Field("pwd") String pwd);
}

3.2 创建接口文件

        Retrofit retrofit = new Retrofit.Builder().baseUrl(Constants).addConverterFactory(GsonConverterFactory.create()).build();
//获取UserMgrService对象 这里其实用到的是动态代理的原理
  UserMgrService userMgrService = retrofit.create(UserMgrService.class);

3.3 返回Call对象

 //调用登陆login方法   注意，这里返回的UserInfoModel类
        final Call<UserInfoModel> call = userMgrService.login("张三", "123456");

3.4 发送异步请求（自动回调）

  //异步请求,enqueue
     call.enqueue(new Callback<UserInfoModel>() {
            @Override
            public void onResponse(Call<UserInfoModel> call, Response<UserInfoModel> response) {
                 int id=response.body().getData().id;
            }
            @Override
            public void onFailure(Call<UserInfoModel> call, Throwable t) {

            }
        });

3.5.怎么动态配置URL地址？

@Path 

public interface IpServiceForPath {
    @GET("{path}/getIpInfo.php?ip=59.108.54.37")  //这里的path对应上面url中的path
    Call<IpModel> getIpMsg(@Path("path") String path);
}

3.6.怎么动态指定查询条件

@Query,就是添加参数

 //get请求，参数用Query，@QueryMap是Query的集合
    @GET("login")   //192.168.106:8080/login?username=张三&pwd=123456,对应baseUrL后面的
    Call<UserInfoModel> login(@Query("username") String username,@Query("pwd") String pwd);

动态指定查询条件组：@QueryMap ，类似于集合

public interface IpServiceForQueryMap {
    @GET("getIpInfo.php")
    Call<IpModel> getIpMsg(@QueryMap Map<String, String> options);
}

4.POST请求

4.1 传键值对

public interface IpServiceForPost {
    @FormUrlEncoded //表单请求
    @POST("getIpInfo.php")
    Call<IpModel> getIpMsg(@Field("ip") String first);
}

4.2 传JSON字符串：@Body

public interface IpServiceForPostBody {
    @POST("getIpInfo.php")
    Call<IpModel> getIpMsg(@Body Ip ip);
}

建立标识对象

public class Ip {
    private String ip;  //只需要1个String对象就行了
    public Ip(String ip) {
        this.ip = ip;
    }
}

        Call<User> updateUser(@Part MultipartBody.Part photo, @Part("descirption")RequestBody description);
