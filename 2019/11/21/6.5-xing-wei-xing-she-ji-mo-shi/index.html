6.5 行为型设计模式

1.策略模式

定义一系列的算法，把每一个算法封装起来，并且使它们可相互替换。策略模式使得算法可独立于使用它的客户而独立变化。 可以解决if..else重复嵌套的问题.核心其实就是把if..else中的每个方法封装起来，为一个个单独类

场景分析：

张无忌作为一个大侠会遇到很多对手，如果每遇 到一个对手他都用自己最厉害的武功去应战，这显然是不明智的。于是张无忌想出了3种应战的策略，分别对付3个实力层次的对手。

1.1 策略接口



/**
 * 策略接口
 */
public interface FightingStrategy {
    public void fighting();
}

1.2 具体策略实现


/**
 * 具体策略实现
 */
public class CommonRivalStrategy implements FightingStrategy {
    @Override
    public void fighting() {
        System.out.println("遇到普通对手");
    }
}


/**
 * 具体策略实现
 */
public class WeakRivalStrategy implements FightingStrategy {
    @Override
    public void fighting() {
        System.out.println("遇到弱对手");
    }
}

1.3 上下文角色



/**
 * 上下文角色
 */
public class Context {
    private FightingStrategy fightingStrategy;

    public Context(FightingStrategy fightingStrategy) {
        this.fightingStrategy = fightingStrategy;
    }

    public void fighting(){
        fightingStrategy.fighting();
    }
}

1.4 客户端调用

         //遇到弱敌
        Context context = new Context(new WeakRivalStrategy());
        context.fighting();
        //遇到普通敌人
        context = new Context(new CommonRivalStrategy());
        context.fighting();

1.5 使用场景

  • 对客户隐藏具体策略（算法）的实现细节，彼此完全独立。
  • 针对同一类型问题的多种处理方式，仅仅是具体行为有差别时。

1.6 优点和缺点

1.6.1 优点

  • 可以避免使用多重条件语句
  • 易于拓展，当需要添加策略，只需要实现接口就行了

1.6.2 缺点

  • 策略的复用性小
  • 与迪米特原则违背，上层模块需要知道有什么策略，才能使用策略

2.模板方法模式

2.1 什么是模板方法模式

定义一个操作中的算法框架，而将一些步骤延迟到子类中，使得子类不改变一个算法的结构即可重定义算法的某些特定步骤。 实际开发中，有些步骤可以变，有些不能变，所以就需要用到模板方法模式。

2.2 简单实现

接着举武侠的例子。一个武侠要战斗的时候，也有一套固定的通用模式，那就是运行内功、开启经脉、准备武器和使用招式，我们把这些用代码表示，如下所示

2.2.1 创建抽象类，定义算法框架

package javaSmallHomeWork1;

/**
 * 抽象类，算法模板框架
 */
public abstract class AbstractSwordsman {
    public final void finghting() {
        //内功，抽象方法
        neigong();
        //调节经脉，具体方法
        meridian();
        if (hasWeapons()) {
            //使用武器
            weapons();
        }
        //钩子方法
        hook();
    }

     void hook() {
    }

    protected abstract void weapons();

    protected  boolean hasWeapons() {
        return true;
    }

    protected void meridian() {
        System.out.println("开启经脉");
    }

    protected abstract void neigong();
}

具体方法是一些统一要执行的步骤

抽象方法是必须延迟到子类实现的方法

钩子方法是有2类，1类是可以视情况决定是否要覆盖的方法，第2类是用于对条件进行判断

2.2.2 具体实现类


public class ZhangSanFeng extends AbstractSwordsman {

    @Override
    protected void weapons() {
        System.out.println("运行纯阳无极棍");
    }

//抽象方法
    @Override
    protected void neigong() {
        System.out.println("运行张山峰神功");
    }

    //钩子方法
    @Override
    protected void hook() {
        System.out.println("去厕所");
    }
}

package javaSmallHomeWork1;

public class ZhangWuJi extends AbstractSwordsman {

    @Override
    protected void weapons() {

    }

    @Override
    protected void neigong() {
        System.out.println("运行九阳神功");
    }

    //钩子方法，决定条件的执行
    @Override
    protected boolean hasWeapons() {
        return false;
    }
}

2.2.3 客户端调用

        ZhangWuJi zhangWuJi=new ZhangWuJi();
        zhangWuJi.finghting();
        ZhangSanFeng zhangSanFeng=new ZhangSanFeng();
        zhangSanFeng.finghting();

2.3 使用场景

  • 逻辑基本相同
  • 可以把核心算法设计为模板方法
  • 实现子类对父类的反向控制

2.4 优点和缺点

2.4.1 优点

  • 不变的行为搬移到超类，去除了子类中的重复代码。
  • 提高了拓展性

2.4.2 缺点

  • 类的个数的增加，设计更加抽象

3.观察者模式

3.1 定义

对象间一种一对多的依赖关系，每当一个对象改变状态时，则所有依赖于它的对象都会得到通知并被自动更新

3.2 简单使用

3.2.1 抽象观察者

我们可以拿微信公众号来举例。假设微信用户就是观察者，微信公众号是被观察者，有多个微信用户关注了“程序猿”这个公众号，当这个公众号更新时就会通知这些订阅的微信用户。


/**
 * 抽象观察者
 */
public interface Observer {
    public void update(String msg);
}

3.2.2 具体观察者


/**
 * 具体观察者
 */
public class WeixinUser implements Observer {
    private String name;

    public WeixinUser(String name) {
        this.name = name;
    }

    @Override
    public void update(String msg) {
        System.out.println(name + "-" + msg);
    }
}

3.2.3 抽象被观察者


/**
 * 抽象被观察者
 */
public interface Subject {
    /**
     * 增加订阅者
     *
     * @param observer 订阅者
     */
    public void attach(Observer observer);

    /**
     * 分离订阅者
     *
     * @param observer
     */
    public void detach(Observer observer);

    /**
     * 通知订阅者更新消息
     *
     * @param msg
     */
    public void notify(String msg);
}

3.2.4 具体被观察者


/**
 * 具体被观察者
 */
public class SubscriptionSubject implements Subject {
    //存储订阅公众号的微信用户，存储观察者
    private List<Observer> weixinUserList = new ArrayList<>();

    @Override
    public void attach(Observer observer) {
        weixinUserList.add(observer);
    }

    @Override
    public void detach(Observer observer) {
        weixinUserList.remove(observer);
    }

    @Override
    public void notify(String msg) {
        for (Observer observer : weixinUserList) {
            observer.update(msg);
        }
    }
}

3.2.5 客户端调用

    public static void main(String[] args) {
        SubscriptionSubject subscriptionSubject = new SubscriptionSubject();
        //创建观察者
        WeixinUser weixinUser1 = new WeixinUser("1");
        WeixinUser weixinUser2 = new WeixinUser("2");
        //订阅，观察者绑定被观察者
        subscriptionSubject.attach(weixinUser1);
        subscriptionSubject.attach(weixinUser2);
        subscriptionSubject.notify("更新了");
    }
