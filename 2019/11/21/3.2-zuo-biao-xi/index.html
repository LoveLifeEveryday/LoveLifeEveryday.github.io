3.2 坐标系

3.2.1 Android坐标系

以屏幕左上角的顶点作为Android的原点，这个原点向右是X轴正方向，向下是Y 轴正方向

在触控事件中，getRawX和getRawY也是Android坐标系的坐标

3.3 View的滑动

1.基本思想是什么？

  1. 记下触摸点的坐标
  2. 移动之后，记下移动后的坐标
  3. 算出偏移量

2.滑动方法有哪些？

3.3.1 layout（）方法

public boolean onTouchEvent(MotionEvent event) {
        //获取到手指处的横坐标和纵坐标
        int x = (int) event.getX();
        int y = (int) event.getY();

        switch (event.getAction()) {
            case MotionEvent.ACTION_DOWN:
                //lastX是存储上一次的x
                lastX = x;
                lastY = y;
                break;

            case MotionEvent.ACTION_MOVE:
                //计算移动的距离
                int offsetX = x - lastX;
                int offsetY = y - lastY;
                //调用layout方法来重新放置它的位置,左上右下
               layout(getLeft()+offsetX, getTop()+offsetY,
                       getRight()+offsetX , getBottom()+offsetY);
                break;
                return true;
        }

3.3.2 offsetLeftAndRight（）与offsetTopAndBottom（）

类似于layout(),将layout方法换为offsetLeftAndRight和offsetTopAndBottom

           // 对left和right进行偏移
           offsetLeftAndRight(offsetX);
            //对top和bottom进行偏移
           offsetTopAndBottom(offsetY);

3.3.3 使用LayoutParams（改变布局参数）

                //使用LayoutParams
     LinearLayout.LayoutParams layoutParams= (LinearLayout.LayoutParams) getLayoutParams();
                //改变Margin
               layoutParams.leftMargin = getLeft() + offsetX;
                layoutParams.topMargin = getTop() + offsetY;
                setLayoutParams(layoutParams);

注意的是：这里的LayoutParams需要根据父控件获得，

如果父控件是RelativeLayout， 则要使用RelativeLayout.LayoutParams

除了使用布局的LayoutParams外，我们还可以用 ViewGroup.MarginLayoutParams来实现：

 //使用MarginLayoutParams
      ViewGroup.MarginLayoutParams layoutParams = (ViewGroup.MarginLayoutParams) getLayoutParams();               layoutParams.leftMargin = getLeft() + offsetX;
               layoutParams.topMargin = getTop() + offsetY;
              setLayoutParams(layoutParams);

3.3.4 动画

1.新建anim文件夹，在里面新建translate.xml
<?xml version="1.0" encoding="utf-8"?>
<set xmlns:android="http://schemas.android.com/apk/res/android"
    android:fillAfter="true">

    <translate
        android:duration="1000"
        android:fromXDelta="0"
        android:toXDelta="300" />
</set>

注意，这里加上fillAfter=”true”是表明停留在动画移动后的位置

2.调用动画
mCustomView.setAnimation(AnimationUtils.loadAnimation(this,R.anim.translate));
3.需要注意什么？

动画并不能改变View的具体位置，只是改变了他的显示位置，所以我们如果在新的位置点击的话，不会有反应

解决方式是用属性动画，既可以执行动画，又可以改变位置

3.3.5 scrollTo与scollBy

scrollTo（x，y）表示移动到一个具体的坐标点

scrollBy（dx，dy）则表示移动的增量为dx、dy,其实底层还是调用scrollTo的

1.scrollTo的源码是怎样的？

2.怎么使用呢？

我们将 ACTION_MOVE中的代码替换成如下代码；

//使用scrollBy
                ((View)getParent()).scrollBy(-offsetX,-offsetY);

为啥这里是偏移是负数呢？

因为这个偏移是把手机屏幕偏移了，界面自然就反方向移动

3.3.6 Scroller

1.为什么要用这个呢

实现有过渡效果的滑动

2.怎么使用呢？

  1. 初始化

    mScroller = new Scroller(context);

  2. 重写computeScroll方法

    当系统调用draw的时候会调用这个方法

    @Override
    public void computeScroll() {
        super.computeScroll();
        if(mScroller.computeScrollOffset()){
            ((View) getParent()).scrollTo(mScroller.getCurrX(),mScroller.getCurrY());
            //通过不断的重绘不断的调用computeScroll方法
            invalidate();
        }
    }

  3. 新建一个smoothScrollTo方法

    public void smoothScrollTo(int destX,int destY){
        int scrollX=getScrollX();
        int delta=destX-scrollX;
        //1000秒内滑向destX
        mScroller.startScroll(scrollX,0,delta,0,2000);
        invalidate();
    }

  4. 调用

    mCustomView.smoothScrollTo(-400,0);//向右平移400像素
