3.5 解析Scroller

什么是Scroller呢？他的工作流程是怎样的呢？

1.什么是Scroller？

实现有过渡效果的滑动

2.startScroller（）源码是怎样的呢？

public void startScroll(int startX, int startY, int dx, int dy, int duration) {
        mMode = SCROLL_MODE;
        mFinished = false;
        mDuration = duration;
        mStartTime = AnimationUtils.currentAnimationTimeMillis();
        mStartX = startX;
        mStartY = startY;
        mFinalX = startX + dx;
        mFinalY = startY + dy;
        mDeltaX = dx;
        mDeltaY = dy;
        mDurationReciprocal = 1.0f / (float) mDuration;
    }

可以看出，startScroller只是用来做前期准备的，并不能使 View 进行滑动

3.computeScroll（）方法是怎样的呢？

computeScroll是什么？

其实就是通过这个方法进行不断地重绘，从而实现滑动效果,重绘的话，会调用draw,draw会调用computeScroll

@Override
    public void computeScroll() {
        super.computeScroll();
        if(mScroller.computeScrollOffset()){
            ((View) getParent()).scrollTo(mScroller.getCurrX(),mScroller.getCurrY());
            //通过不断的重绘不断的调用computeScroll方法
            invalidate();
        }
    }

4.在scoller中怎么获取当前的ScrollX和ScrollY呢？

其实就是在调用scrollTo前，调用computeScrollOffset,

computeScrollOffset（）源码：

public boolean computeScrollOffset() {
        if (mFinished) {
            return false;
        }

    //计算持续时间
        int timePassed = (int)(AnimationUtils.currentAnimationTimeMillis() - mStartTime);

    //小于设定时间，还没结束
        if (timePassed < mDuration) {
            switch (mMode) {
                    //执行
            case SCROLL_MODE:
                final float x = mInterpolator.getInterpolation(timePassed * mDurationReciprocal);
            //得到当前的位置
                    mCurrX = mStartX + Math.round(x * mDeltaX);
                    mCurrY = mStartY + Math.round(x * mDeltaY);
                break;

            case FLING_MODE:
                final float t = (float) timePassed / mDuration;
                final int index = (int) (NB_SAMPLES * t);
                float distanceCoef = 1.f;
                float velocityCoef = 0.f;
                if (index < NB_SAMPLES) {
                    final float t_inf = (float) index / NB_SAMPLES;
                    final float t_sup = (float) (index + 1) / NB_SAMPLES;
                    final float d_inf = SPLINE_POSITION[index];
                    final float d_sup = SPLINE_POSITION[index + 1];
                    velocityCoef = (d_sup - d_inf) / (t_sup - t_inf);
                    distanceCoef = d_inf + (t - t_inf) * velocityCoef;
                }

                mCurrVelocity = velocityCoef * mDistance / mDuration * 1000.0f;

                mCurrX = mStartX + Math.round(distanceCoef * (mFinalX - mStartX));
                // Pin to mMinX <= mCurrX <= mMaxX
                mCurrX = Math.min(mCurrX, mMaxX);
                mCurrX = Math.max(mCurrX, mMinX);

                mCurrY = mStartY + Math.round(distanceCoef * (mFinalY - mStartY));
                // Pin to mMinY <= mCurrY <= mMaxY
                mCurrY = Math.min(mCurrY, mMaxY);
                mCurrY = Math.max(mCurrY, mMinY);

                if (mCurrX == mFinalX && mCurrY == mFinalY) {
                    mFinished = true;
                }

                break;
            }
        }
        else {
            mCurrX = mFinalX;
            mCurrY = mFinalY;
            mFinished = true;
        }
        return true;
    }

首先会计算动画持续时间，如果持续时间小于设定的滑动时间的话，就执行Switch,因为在startScroll中的mMode值为SCROLL_MODE，执行分支语句 SCROLL_MODE，然后得到滑动后的位置

5.总结

其实就是Scroller不能直接实现滑动，先start,再调用重绘invalidate，重绘调用draw，draw调用computeScroll，computeScroll中调用computeScrollOffset获取当前位置，再调用invalidate，循环进行，直到动画结束
