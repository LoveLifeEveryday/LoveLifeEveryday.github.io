6.4 结构型设计模式

什么是结构型设计模式？其实结构型设计模式是为了解决模块间的耦合问题

1.代理模式

定义：为其他对象提供一种代理以控制对这个对象的访问。

1.1 静态代理的写法

1.1.1 抽象主题类

public interface Subject {
    /**
     * 接口方法
     */
    public void request();
}

1.1.2 真实主题

public class ConcreteSubject implements Subject {
    /**
     * 具体的业务逻辑实现
     */
    @Override
    public void request() {
        //业务处理逻辑
    }
}

1.1.3 代理类

public class Proxy implements Subject {

    /**
     * 要代理的实现类
     */
    private Subject subject = null;

    /**
     * 默认代理自己
     */
    public Proxy() {
        this.subject = new Proxy();
    }

    public Proxy(Subject subject) {
        this.subject = subject;
    }

    /**
     * 构造函数，传递委托者
     *
     * @param objects 委托者
     */
    public Proxy(Object... objects) {
    }

    /**
     * 实现接口方法
     */
    @Override
    public void request() {
        this.before();
        this.subject.request();
        this.after();
    }

    /**
     * 预处理
     */
    private void before() {
        //do something
    }

    /**
     * 后处理
     */
    private void after() {
        //do something
    }
}

1.1.4 客户端类

public class Client {
    public static void main(String[] args) {
        Subject subject = new ConcreteSubject();
        Proxy proxy = new Proxy(subject);
        proxy.request();
    }
}

1.2 动态代理的写法

1.2.1 动态代理类


/**
 * 动态代理
 * created by xucanyou666
 * on 2019/11/14 20:08
 * email：913710642@qq.com
 * @author xucanyou666
 */

public class DynamicPurchasing implements InvocationHandler {
    private Object object;

    public DynamicPurchasing(Object o) {
        object = o;
    }

    /**
     * 动态代理
     *
     * @param proxy
     * @param method
     * @param args
     * @return
     * @throws Throwable
     */
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        Object result = method.invoke(object, args);
        //方法名等于buy
        if (method.getName().equals("buy")) {
            Log.d(TAG, "买");
        }
        return result;
    }

    private static final String TAG = "DynamicPurchasing";
}

1.2.2 客户端类

package com.users.xucanyou666.testdashboard;
/**
 * created by xucanyou666
 * on 2019/11/14 20:12
 * email：913710642@qq.com
 */
public class Client {
    public static void main(String[] args) {
        //被代理类，真实对象
        Lenovo lenovo = new Lenovo();
        //动态代理类
        DynamicPurchasing dynamicPurchasing = new DynamicPurchasing(lenovo);
        //类加载器
        ClassLoader loader = lenovo.getClass().getClassLoader();
        //由动态代理生成代理类对象
        Lenovo mLenovo = (Lenovo) Proxy.newProxyInstance(loader, new Class[]{Lenovo.class}, dynamicPurchasing);
        mLenovo.start();
    }
}

1.3 代理模式的种类

  • 远程代理：为一个对象在不同的地址空间提供局部代表，这样系统可以将 Server 部分的实现隐藏
  • 虚拟代理：使用一个代理对象表示一个十分耗费资源的对象并在真正需要时才创建。
  • 安全代理：用来控制真实对象访问时的权限。一般用于真实对象有不同的访问权限时。
  • 智能指引：代理处理另外一些事，比如计算真实对象的引用计数

1.4 代理模式的优点和缺点

1.4.1 优点

  • 实现实际的业务逻辑，不用关心其他非本职的工作。 职责单一
  • 真实主题类随时都会发生变化；但是因为它实现了公共的接口，所以代理类可以不做任何修改就能够
    使用。

1.4.2 缺点

它始终无法摆脱仅支持 interface代理的桎梏

2.装饰模式

2.1 是什么？

继承的替代方案之一。它通过创建一个包装对象，也就是装饰来包裹真实的对象。

2.2 写法

4.享元模式

池技术的重要实现方式，

定义：使用共享对象有效地支持大量细粒度的对象

要求细粒度对象，那么不可避免地使得对象数量多且性质相近。这些对象分为两个部分：内部状态和外部状态。

内部状态：对象不会变的信息

外部状态：对象会变的信息

4.1 实现

4.1.1 抽象享元角色

package com.users.xucanyou666.testdashboard;

/**
 * 抽象享元角色 这里用的是外部状态的变量
 * created by xucanyou666
 * on 2019/11/14 21:41
 * email：913710642@qq.com
 */
public interface IGoods {
    public void showGoodsPrice(String name);
}

4.1.2 具体享元角色

package com.users.xucanyou666.testdashboard;


import android.util.Log;

/**
 * 具体享元角色
 * created by xucanyou666
 * on 2019/11/14 21:42
 * email：913710642@qq.com
 */
public class Goods implements IGoods {
    private String name;//名称 内部状态
    private String version;//版本 外部状态

    Goods(String name) {
        this.name = name;
    }

    private static final String TAG = "Goods";

    @Override
    public void showGoodsPrice(String version) {
        if (version.equals("32G")) {
            Log.d(TAG, "价格为5199");
        } else if (version.equals("128G")) {
            Log.d(TAG, "价格为5999");
        }
    }
}

4.1.3 享元工厂



/**
 * 享元工厂
 * created by xucanyou666
 * on 2019/11/14 21:45
 * email：913710642@qq.com
 */
public class GoodsFactory {
    //池
    private static Map<String, Goods> pool = new HashMap<>();

    public static Goods getGoods(String name) {
        //这里的匹配是内部状态
        if (pool.containsKey(name)) {
            //使用缓存
            return pool.get(name);
        } else {
            Goods goods = new Goods(name);
            pool.put(name, goods);
            return goods;
        }
    }
}

4.1.4 客户端调用

 Goods goods=GoodsFactory.getGoods("iphone7");
       goods.showGoodsPrice("32G");
       Goods goods1=GoodsFactory.getGoods("iphone7");
       goods.showGoodsPrice("32G");
